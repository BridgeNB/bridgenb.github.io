Java Abstract class and Interfaces are two mechanisms to define abstract concepts. Thanks to these two guys, Java can have such strong in Object-oriented language. Abstract class and interfaces are very similar but different. Let us go through details below.

Abstract class

Every thing in Java is an object described by classes. However, not every class is for specifing object but some are to help to make object more through, which we called Abstract class. For example

This is a class to create an animal object, but the specific content of this animal is unknown yet, which is an good example of abstract class.
Since abstract class is a class, so it can be inherited by other class to form polymorphism. Below are the points to be aware of to implement abstract class:

  • abstract class cannot be instantiated; instantiations must be done by its children class.
  • abstract methods must be rewriten by its children class.
  • if there is one abstract method in that class, it must be defined as an abstract method.
  • abstract class may have instantiated methods, and it can also be without abstact methods.
  • The childern class cannot have a abstract method with same name as father’s abstract method
  • abstact cannot work with final, private, static, and native to describe same methods.

Inteface

Interfaces are kind of protocol builded among classes; they are the extension of abstract classes. However, interfaces are not classes; they cannot be instantiated like classes. A Java class could implement multiple interfaces but extends only one class. Interfaces are for multiple “inheritances” and keep data safe at the same time.
There are severl points to aware of to construct interfaces.

  • All interface would be public
  • if a non abstract class implemented that interface, this class must implement all mothods inside of that interface. On the contrary, the abstract class do not need to implement all methods when it implement an interface.
  • Must not have same methods name when implements multiple interfaces.

Difference between Abstract Class and Interface from design aspect

The essiential difference between abstract class and interfaces are in the design aspect.

  • Difference in abstract level:
    Abstract class is class abstraction; Inteface is action abstration.
    • Abstract class abstracted everything of a class, including property and action, however interface is just an abstration to a class’s action.
  • Difference in range:
    Abstract class is for a range of similar classes, such as Bicycle, Car classes can have a same animal class, since abstract class finds common points from childern classes, and pull those common points to an abstract class. However, interface is quite different, the classes implement same interface can be total different. For example, Bicycle and Weed can have same Roll action, but they are totally differenct that cannot be abstact as one class.
    • In conclusion, abstract class must have something in common (is-a relation) however, interface do not need to have this kind of relationship. Simpily, if two classes have same action, they are allowed to implement same interface.
  • Difference in design
    Abstract class must know common points of all its childern class. Interface do need common points for classes who implement it. So interfaces can be used for totally different class with same action.

In conclusion

  1. One class can only extend on abstract class but multiple interfaces
  2. There are a number of differences between interface and abstract class in design level.
